# 2026-06-27 ref 实现原理

## 今天学了什么

- ref 如何让基本类型也有响应式
- getter/setter 拦截 .value 的读写
- ref 包裹对象时的双层代理机制
- ref vs reactive 的使用场景

## 核心代码

### 迷你 ref 实现

```javascript
export default function ref(rawVal) {
  const depMap = new Map()
  let val = rawVal

  // 对象类型用 reactive 包裹
  if (rawVal instanceof Object) {
    val = reactive(rawVal)
  }

  return {
    get value() {
      track(depMap, 'value')
      return val
    },
    set value(newVal) {
      val = newVal
      trigger(depMap, 'value')
    }
  }
}
```

### track / trigger（从 reactive 抽离）

```javascript
export function track(depMap, key) {
  if (currentEffect) {
    if (!depMap.has(key)) {
      depMap.set(key, new Set())
    }
    depMap.get(key).add(currentEffect)
  }
}

export function trigger(depMap, key) {
  if (depMap.has(key)) {
    depMap.get(key).forEach(fn => {
      fn.scheduler ? fn.scheduler() : fn()
    })
  }
}
```

## 关键理解

### ref 的本质

```
基本类型 → getter/setter 直接拦截值
对象类型 → getter/setter + reactive 双层代理
```

### 依赖收集时机

```javascript
effect(() => {
  console.log(refState.value.name)
})

// 执行过程：
// 1. ref.getter → track(ref 的 dep) → 收集 effect
// 2. 返回 reactive 代理
// 3. reactive.getter name → track(name 的 dep) → 收集 effect
```

### ref vs reactive 的区别

| 特性 | ref | reactive |
|------|-----|----------|
| 基本类型 | ✅ 支持 | ❌ 不支持 |
| 访问方式 | `.value` | 直接访问 |
| 替换整个对象 | `state.value = {}` ✅ | ❌ 会丢失响应式 |
| 依赖粒度 | `.value` 共享一个 dep | 每个属性独立 dep |
| 性能 | 多一层 getter | 更直接 |

### 依赖收集验证

```javascript
const refState = ref({ name: 'Vue', count: 0 })

effect(() => {
  console.log('只监听 name:', refState.value.name)
})

refState.value.count = 999  // ❌ 不触发（effect 没收集到 count 的 dep）
refState.value = { name: 'x' }  // ✅ 触发（ref 的 dep）
```

**结论**：effect 只会监听实际访问的属性，不会因为 ref 的存在而收集所有属性

## 使用建议

```javascript
// ✅ 推荐
const count = ref(0)            // 基本类型用 ref
const state = reactive({ ... }) // 对象用 reactive

// ⚠️ 不推荐
const state = ref({ ... })      // 多一层代理，没必要
```

## Vue 源码对比

| 方面 | 我们的实现 | Vue 的实现 |
|------|-----------|-----------|
| 存储 | let val | this._rawValue / this._value |
| 对象处理 | reactive(rawVal) | toReactive(rawVal) |
| 依赖收集 | track(depMap, 'value') | trackEffects(this.dep) |
| 触发更新 | trigger(depMap, 'value') | triggerEffects(this.dep) |

## 踩坑记录

1. **val 声明遗漏**：忘记用 let 声明，变成全局变量
2. **trigger 顺序**：必须先赋值再触发，否则 effect 拿到旧值
3. **ESM 导入**：Node.js 需要完整的 `.js` 扩展名
4. **依赖收集时机**：只在 effect 执行期间才会 track

## 面试话术

> "ref 的核心是用 getter/setter 拦截 .value 的读写。读取时 track 收集依赖，设置时先赋值再 trigger 触发更新。对于对象类型，ref 内部会调用 reactive 进行二次代理，所以访问 ref 对象时会有两层拦截。实际使用建议：基本类型用 ref，对象用 reactive，避免不必要的性能开销。"
