# 2026-06-21 阶段2：响应式系统核心原理

## 今天学了什么

- reactive() 的实现原理：Proxy 拦截 get/set
- effect() 的实现原理：全局变量记录当前 effect
- track 依赖收集：get 时把当前 effect 加入依赖集合
- trigger 触发更新：set 时执行所有依赖该属性的 effect
- 手写了迷你响应式系统，与 Vue 源码做了对比

## 关键代码片段

### 手写的迷你响应式系统（mini-reactive.js）

```javascript
let currentEffect = null

function effect(fn) {
  currentEffect = fn
  fn()
  currentEffect = null
}

function reactive(obj) {
  const depMap = new Map()
  return new Proxy(obj, {
    get(target, key) {
      if (currentEffect) {
        if (!depMap.has(key)) {
          depMap.set(key, new Set())
        }
        depMap.get(key).add(currentEffect)
      }
      return target[key]
    },
    set(target, key, newVal) {
      target[key] = newVal
      if (depMap.has(key)) {
        depMap.get(key).forEach(fn => fn())
      }
      return true
    }
  })
}
```

### Vue 源码对比

| 概念 | 我们写的 | Vue 的实现 |
|------|---------|-----------|
| 当前 effect | `currentEffect` | `activeSub` |
| 依赖存储 | `Map<key, Set<effect>>` | `WeakMap<target, Map<key, Dep>>` + 双向链表 |
| 依赖收集 | `depMap.get(key).add(currentEffect)` | `dep.track()` |
| 触发更新 | `depMap.get(key).forEach(fn => fn())` | `dep.trigger()` |

## 踩坑记录

1. Proxy 只接受两个参数：`new Proxy(target, handler)`，不能把 get 和 set 分成两个对象
2. set 必须返回 true，否则会报错
3. set 时要判断 depMap.has(key)，否则修改未监听的属性会报错

## 面试话术

> "Vue 3 的响应式基于 Proxy 实现。reactive() 用 Proxy 包装对象，get 拦截时调用 track() 做依赖收集，set 拦截时调用 trigger() 触发更新。effect() 用全局变量 activeSub 记录当前正在执行的副作用函数，这样 track 知道'谁在用这个数据'。依赖关系用 WeakMap + Map + 双向链表存储，相比 Vue 2 的 Object.defineProperty，Proxy 可以监听新增属性和数组变化。"
