# 2026-06-26 阶段2（续）：watch 侦听器实现原理

## 今天学了什么

- watch 的核心原理：effect 收集依赖 + scheduler 触发 callback
- watch 的 immediate 选项：创建时立即执行一次
- watch 的 deep 选项：traverse 递归遍历收集依赖
- reactive 的 Proxy 缓存机制（WeakMap）

## 关键代码片段

### 迷你 watch 完整实现

```javascript
export default function watch(source, callback, { immediate = false, deep = false } = {}) {
  let oldVal = update(source)

  effect(() => {
    if (deep) {
      traverse(update(source))  // 深度遍历，收集所有嵌套属性的依赖
    } else {
      update(source)            // 只收集第一层依赖
    }
  }, {
    scheduler: () => {
      let newVal = update(source)
      callback(newVal, oldVal)  // 传入新值和旧值
      oldVal = newVal           // 更新旧值
    }
  })

  if (immediate) {
    callback(oldVal, undefined)  // 立即执行，oldVal 为 undefined
  }
}

function update(source) {
  return typeof source === "function" ? source() : source
}

function traverse(obj) {
  if (typeof obj !== "object" || obj === null) {
    return
  }
  for (let key in obj) {
    traverse(obj[key])  // 递归遍历，访问属性触发 get 收集依赖
  }
  return obj
}
```

### reactive 的 Proxy 缓存

```javascript
const proxyCache = new WeakMap()

export function reactive(obj) {
  // 如果已经创建过 Proxy，直接返回缓存
  if (proxyCache.has(obj)) {
    return proxyCache.get(obj)
  }

  const depMap = new Map()
  const proxy = new Proxy(obj, {
    get(target, key) {
      if (currentEffect) {
        if (!depMap.has(key)) {
          depMap.set(key, new Set())
        }
        depMap.get(key).add(currentEffect)
      }
      const value = target[key]
      // 嵌套对象也要变成响应式
      if (typeof value === "object" && value !== null) {
        return reactive(value)
      }
      return value
    },
    set(target, key, newVal) {
      target[key] = newVal
      if (depMap.has(key)) {
        depMap.get(key).forEach(fn => {
          if (fn.scheduler) {
            fn.scheduler()
          } else {
            fn()
          }
        })
      }
      return true
    }
  })

  proxyCache.set(obj, proxy)  // 缓存
  return proxy
}
```

## 核心原理对比

| 概念 | computed | watch |
|------|----------|-------|
| 目的 | 计算一个新值 | 执行副作用（callback） |
| 触发时机 | 读取 .value 时 | 依赖变化时 |
| 返回值 | 返回计算结果 | 无返回值 |
| 核心实现 | effect + scheduler + 缓存 | effect + scheduler + callback |

## Vue 源码对比

| 功能 | 我们写的 | Vue 的实现 |
|------|---------|-----------|
| source 处理 | typeof 判断函数 | isRef/isReactive/isFunction 多种类型 |
| deep 实现 | traverse 递归遍历 | traverse + depth 参数控制深度 |
| immediate | if (immediate) callback() | job 函数里判断 immediateFirstRun |
| cleanup | 未实现 | onWatcherCleanup 注册清理函数 |

## 踩坑记录

1. **source 覆盖问题**：`source = source()` 会覆盖原函数，导致后续调用失败
2. **effect 里不调用 source**：依赖收集不到，scheduler 永远不会触发
3. **嵌套对象不是 Proxy**：get trap 要返回 `reactive(value)` 才能收集深层依赖
4. **Proxy 缓存**：每次访问都创建新 Proxy 会导致依赖分散，用 WeakMap 缓存

## 面试话术

> "watch 的核心是 effect + scheduler。创建 watch 时，effect 执行一次收集依赖；当依赖变化时，scheduler 触发 callback 并传入新旧值。deep 选项通过 traverse 递归遍历对象所有属性来收集深层依赖。Vue 3 的 reactive 用 WeakMap 缓存 Proxy，避免重复创建。"
